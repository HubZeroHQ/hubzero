# Archive

These documents are historical. They record planning proposals and dated point-in-time audits from before the site reached release quality. They are kept for context, not as current guidance.

In particular:

- [`CONTENT_AUDIT.md`](CONTENT_AUDIT.md), [`EDITORIAL_GAP_REPORT.md`](EDITORIAL_GAP_REPORT.md), [`LAUNCH_READINESS.md`](LAUNCH_READINESS.md), and [`PHASE_22_LAUNCH_READINESS.md`](PHASE_22_LAUNCH_READINESS.md) are dated snapshots of Studio content readiness, each explicitly scoped to the data that existed on its audit date. Their go/no-go conclusions describe that snapshot, not the current state of the site.
- [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md) is the phased build roadmap that preceded implementation.
- `screenshots/` contains dated visual evidence captured during past phase reviews.

For the current, living architecture and design references, see the parent [`docs/`](../README.md) directory.
